CSS 样式书写规范

1.给组件书写样式的时候应该命名成一个系列，最多三层。这样在分离的时候可以做为一个整体比较方便
（如果使用了cssmodal插件，会自动帮你打包命名）
2.类的命名要么用驼峰，要么用-分隔，统一风格

Block概念及元素宽高设置

block和inline概念：

block-level elements (块级元素) 和 inline elements (内联元素)。
block元素通常被现实为独立的一块，会单独换一行；inline元素则前后不会产生换行，一系列inline元素都在一行内显示，直到该行排满。

常见的块级元素有 DIV, FORM, TABLE, P, PRE, H1~H6, DL, OL, UL 等。
常见的内联元素有 SPAN, A, STRONG, EM, LABEL, INPUT, SELECT, TEXTAREA, IMG, BR 等。

块级元素：

除了absolute,fixed,浮动元素之外，其他的块级元素，宽度是从父元素继承的（自动填满父元素的宽度）
块级元素设置了宽高仍然可以独占一行，可以设置margin和padding属性
一般不会给块级元素设置为行内块，当块级元素需要一行显示多个的时候，会设置float

行内元素（span/i）：

宽度由子元素宽度决定，而且不能手动设置宽高，水平方向的margin和padding会产生效果，但是垂直方向不会产生效果
可以通过display:inline-block来把行内元素，设置为行内块，以便于设置宽高（伪元素/span元素设置背景图）

布局的基础概念

定位的基础概念

相对定位：

就是相对于文档中的其他元素进行定位：
那些脱离了文档流的元素，比如设置了浮动或者绝对定位的元素，不会对相对定位产生影响
relative和static都是相对于文档其他元素进行定位，都属于相对定位的范畴，区别就在与一个可以控制位移一个不能

static

默认值，任意 position: static; 的元素不会被特殊的定位。一个 static 元素表示它不会被“positioned”。
Top,left,right,bottom,z-index都不会生效。

relative

表现的和 static 一样，除非你添加了一些额外的属性。
在一个相对定位（position属性的值为relative）的元素上设置 top、 right 、 bottom 和 left 属性会使其偏离其正常位置。其
他的元素则不会调整位置来弥补它偏离后剩下的空隙。

可以用position:relative和top、left的设置来实现元素位置的微调

绝对定位：

绝对定位的特点：

块级元素的宽度在未定义时不再为100%，而是根据内容自动调整
在不定义z-index的情况下，absolute元素会覆盖在其他元素之上
它会脱离正常的文档流，不在占据空间，类似于浮动后的效果

Fixed

一个固定定位（position属性的值为fixed）元素会相对于视窗来定位，这意味着即便页面滚动，它还是会停留在相同的位置。和 relative 一样， top 、 right 、 bottom和 left 属性都可用。
我相信你已经注意到页面右下角的固定定位元素。你现在可以仔细看看它，这里有它所使用的CSS：

#####absolute 是最棘手的position值。
absolute 与 fixed 的表现类似，除了它不是相对于视窗而是相对于最近的“positioned”祖先元素。如果绝对定位（position属性的值为absolute）的元素没有“positioned”祖先元素，那么它是相对于文档的 body 元素，并且它会随着页面滚动而移动。
记住一个“positioned”元素是指position 值不是 static的元素。
相对于上一个不是static的父元素进行绝对定位，也就是说如果不指定父元素的position,absolute将会相对于整个html文档进行绝对定位

浮动的概念

设置了浮动的元素会脱离正常的文档流，不仅在y轴上浮动，在z轴上也浮动了起来。
父元素的高度会根据子元素的高度自动进行调整，如果我们将子元素设置为浮动，父元素的高度就会变为0

###Rem 和百分比
根字体是根据屏幕的尺寸不同去设置的，例如37.5,32等，就是375的屏幕尺寸，和320的屏幕尺寸除以10计算出来的。
用375px的屏幕宽度为例子，在布局的时候如果给图片宽度设置375px,然后打包工具里面的pxtoRem的方法（写死了转换单位是37.5）就会把元素的尺寸转换成10rem,（10rem*根字体37.5）就占满了正屏幕
我们的设计图是按照375的屏幕尺寸来设计的，所以我们可以这样来给图片设置宽度，然后打包工具打包成rem然后框架内的js会检测到不同屏幕宽度，设置不同的根字体。
浏览器会用转换出的rem去乘以当前的根字体（解析rem转换成px）
比如10rem乘以ipad(100的根字体)所以图片的尺寸就是1000px，就可以自适应屏幕的尺寸。(这样所有的字和图片就会很大)
如果一定要用iphnoe的设计去兼容ipad的设计就需要手工的给ipad的根字体设置成37.5然后在布局上使用百分比的方式，不要外层元素写死一个宽度
